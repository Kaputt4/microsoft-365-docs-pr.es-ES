# <a name="allotment-basics"></a>Conceptos básicos de asignación

Las asignaciones de licencias le permiten establecer límites de licencia y delegar la administración de la asignación de licencias solo en los productos y límites de licencia que seleccione.

Las asignaciones usan licencias basadas en grupos para asignar licencias a los usuarios. Los límites de licencia proporcionan un control agregado sobre cuántas licencias se asignan a los usuarios de los grupos. Por lo tanto, incluso a medida que aumenta el número de usuarios de los grupos, puede asegurarse de que se mantiene dentro del límite de licencia que ha establecido para la asignación.

También puede delegar la administración de las asignaciones. Los propietarios delegados de asignación obtienen acceso al centro de administración, pero solo pueden ver y administrar las licencias en las asignaciones que poseen. Esto proporciona una delegación más granular de la administración de licencias dentro de la organización.

## <a name="prerequisites"></a>Requisitos previos

Debe cumplir los requisitos de licencias para [licencias basadas en grupos.](/azure/active-directory/fundamentals/active-directory-licensing-whatis-azure-portal#licensing-requirements)

Puede usar las asignaciones con cualquier producto disponible para los usuarios:

- Office suites y productos independientes
- Enterprise y productos de movilidad
- Productos de Dynamics 365

Los siguientes productos no se pueden usar con asignaciones:

- Microsoft Store aplicaciones
- Software perpetuo o software que se asigna directamente a un usuario si no hay ninguna licencia implicada.
- Recursos de Azure

Debe ser un administrador global o de licencia para empezar con una asignación.
