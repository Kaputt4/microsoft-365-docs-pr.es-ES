# <a name="microsoft-365-tenant-to-tenant-migrations"></a>Migraciones de inquilino a inquilino de Microsoft 365

Existen varios enfoques de arquitectura para fusiones, adquisiciones, desintecciones y otros escenarios que pueden llevar a migrar un inquilino de Microsoft 365 existente a un nuevo inquilino. La mayoría de los clientes trabajan con servicios de consultoría de Microsoft o un partner de Microsoft para migrar inquilinos, incluido el uso de herramientas de terceros para migrar contenido. 

Use el [modelo de arquitectura](../downloads/Microsoft-365-tenant-to-tenant-migration.pdf) de migración de inquilino a inquilino para comprender cómo planear las migraciones de inquilino a inquilino de Microsoft 365 y los pasos de una migración.

[![Modelo de migración de inquilino a inquilino](../media/solutions-architecture-center/msft-tenant-to-tenant-migration-thumb.png)](../downloads/Microsoft-365-tenant-to-tenant-migration.pdf) 

También puede descargar este modelo en formato [PDF](https://github.com/MicrosoftDocs/microsoft-365-docs/raw/public/microsoft-365/downloads/Microsoft-365-tenant-to-tenant-migration.pdf) o [Visio](https://github.com/MicrosoftDocs/microsoft-365-docs/raw/public/microsoft-365/downloads/Microsoft-365-tenant-to-tenant-migration.vsdx) e imprimirlo en papel carta, legal o tabloide (11 x 17).

Este modelo proporciona instrucciones y un punto de partida para la planeación con secciones sobre:

- Asignación de escenarios empresariales a enfoques de arquitectura
- Consideraciones sobre diseño

Este modelo también contiene ejemplos detallados de:

- Un flujo de migración de eventos único
- Flujo de migración por fases
- Movimiento de inquilinos o flujo dividido
