# <a name="microsoft-365-client-app-support-conditional-access"></a>Compatibilidad con aplicaciones cliente de Microsoft 365: acceso condicional

En el lugar de trabajo moderno, los usuarios pueden obtener acceso a los recursos de su organización con varios dispositivos y aplicaciones desde cualquier lugar. Como resultado, ya no basta con centrarse en quién puede obtener acceso a un recurso. La organización también debe admitir cómo y dónde se tiene acceso a un recurso en la infraestructura de control de acceso.

Con el dispositivo, la ubicación y el acceso condicional basado en autenticación multifactor de Azure Active Directory, puede cumplir este nuevo requisito. El acceso condicional es una funcionalidad de Azure Active Directory que le permite aplicar controles en el acceso a aplicaciones de su entorno, todo ello en función de condiciones específicas y administrado desde una ubicación central.

Obtenga más información [sobre El acceso condicional de Azure Active Directory](/azure/active-directory/conditional-access/).

## <a name="supported-clients--platforms"></a>Clientes compatibles & plataformas

Las versiones más recientes de los siguientes clientes y plataformas admiten el acceso condicional. Para obtener más información acerca de la compatibilidad con plataformas en Microsoft 365, vea Requisitos del sistema [para Microsoft 365](/microsoft-365/microsoft-365-and-office-resources).
