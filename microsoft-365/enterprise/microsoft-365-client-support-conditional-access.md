# <a name="microsoft-365-client-app-support-conditional-access"></a>Microsoft 365 Compatibilidad con aplicaciones cliente: acceso condicional

En el lugar de trabajo moderno, los usuarios pueden obtener acceso a los recursos de su organización con varios dispositivos y aplicaciones desde cualquier lugar. Como resultado, ya no basta con centrarse en quién puede obtener acceso a un recurso. La organización también debe admitir cómo y dónde se tiene acceso a un recurso en la infraestructura de control de acceso.

Con Azure Active Directory, ubicación y acceso condicional basado en autenticación multifactor, puedes cumplir este nuevo requisito. El acceso condicional es una funcionalidad de Azure Active Directory que te permite aplicar controles en el acceso a las aplicaciones de tu entorno, todo en función de condiciones específicas y administrados desde una ubicación central.

Obtenga más información [sobre Azure Active Directory acceso condicional](/azure/active-directory/conditional-access/).

## <a name="supported-clients--platforms"></a>Clientes compatibles & plataformas

Las versiones más recientes de los siguientes clientes y plataformas admiten el acceso condicional. Para obtener más información acerca de la compatibilidad de plataforma Microsoft 365, vea [Requisitos del](/microsoft-365/microsoft-365-and-office-resources)sistema para Microsoft 365 .
