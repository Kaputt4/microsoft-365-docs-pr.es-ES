# <a name="step-5-evaluate-microsoft-defender-for-cloud-apps"></a>Paso 5. Evaluar Microsoft Defender for Cloud Apps

**Se aplica a:**
- Microsoft 365 Defender


En este artículo se describe el proceso para habilitar y Microsoft Defender for Cloud Apps piloto junto con Microsoft 365 Defender. Antes de iniciar este proceso, asegúrese de que ha revisado el proceso general para [evaluar Microsoft 365 Defender](eval-overview.md) y ha [creado el entorno de evaluación de Microsoft 365 Defender](eval-create-eval-environment.md). 
<br>

Siga estos pasos para habilitar y Microsoft Defender for Cloud Apps piloto.

:::image type="content" source="../../media/defender/m365-defender-office-eval-steps.png" alt-text="Los pasos para agregar Microsoft Defender para Office al entorno de evaluación de Defender" lightbox="../../media/defender/m365-defender-office-eval-steps.png":::

|Paso  |Descripción  |
|---------|---------|
|[Revisión de los requisitos de arquitectura y los conceptos clave](eval-defender-mcas-architecture.md)    | Comprenda la arquitectura de Defender for Cloud Apps y cómo se integra con Microsoft 365 Defender, Microsoft Defender para punto de conexión y Azure Active Directory.        |
|[Habilitación del entorno de evaluación](eval-defender-mcas-enable-eval.md)     | Conéctese al portal, configure la integración con Defender for Identity o los dispositivos de red de su organización y comience a ver y administrar aplicaciones en la nube.         |
|[Configuración del piloto ](eval-defender-mcas-pilot.md)    | Establezca el ámbito de la implementación en determinados grupos de usuarios, configure el control de aplicaciones de acceso condicional y pruebe los tutoriales para proteger el entorno.       |
