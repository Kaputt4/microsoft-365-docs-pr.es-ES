# <a name="responding-to-your-first-incident"></a>Respuesta al primer incidente

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender.md)]

**Se aplica a:**
- Microsoft 365 Defender

La estrategia de respuesta a incidentes de una organización determina su capacidad para hacer frente a incidentes de seguridad y cibercrimen cada vez más perturbadores. Aunque es importante tomar medidas preventivas, la capacidad de actuar rápidamente para contener, erradicar y recuperarse de incidentes detectados puede minimizar los daños y las pérdidas empresariales.

En este tutorial de respuesta a incidentes se muestra cómo, como parte de un equipo de operaciones de seguridad (SecOps), puede realizar la mayoría de los pasos clave de respuesta a incidentes dentro de Microsoft 365 Defender. Estos son los pasos:

- Preparación de la posición de seguridad
- Para cada incidente:
  - Paso 1: Evaluación y análisis
  - Paso 2: Corrección (contención, erradicación y recuperación)
  - Paso 3: Revisión posterior al incidente

Un incidente de seguridad es definido por el Instituto Nacional de Estándares y Tecnología (NIST) como "una ocurrencia que en realidad o potencialmente pone en peligro la confidencialidad, integridad o disponibilidad de un sistema de información; o la información que el sistema procesa, almacena o transmite; o que constituye una violación o amenaza inminente de violación de directivas de seguridad, procedimientos de seguridad o directivas de uso aceptable".

Los incidentes en Microsoft 365 Defender son los puntos de partida lógicos para el análisis y la respuesta a incidentes. El análisis y la corrección de incidentes suelen componer la mayoría de las tareas y el tiempo de un equipo (SecOps).
