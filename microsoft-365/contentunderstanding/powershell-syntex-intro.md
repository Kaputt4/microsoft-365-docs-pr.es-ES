# <a name="manage-microsoft-syntex-by-using-powershell"></a>Administración de Microsoft Syntex mediante PowerShell

> [!IMPORTANT]
> Los cmdlets de PowerShell de Microsoft Syntex y todos los demás componentes de PnP son herramientas de código abierto respaldadas por una comunidad activa que proporciona soporte técnico para ellos. Los canales oficiales de soporte técnico de Microsoft no ofrecen ningún contrato de nivel de servicio para herramientas de código abierto.

En estos escenarios, los cmdlets de Syntex del módulo de PowerShell PnP se pueden usar para interactuar con modelos y explicaciones. Para más información sobre cómo instalar este módulo, consulte [Información general de PowerShell de PnP](/powershell/sharepoint/sharepoint-pnp/sharepoint-pnp-cmdlets).

Seleccione uno de los siguientes escenarios para aprender a usar PowerShell para administrar Syntex:

- [**Exportación e importación de modelos**](powershell-syntex-import-export.md)

    Comience aquí si desea obtener información sobre cómo exportar un modelo de procesamiento de documentos no estructurado desde un centro de contenido e importarlo a otro centro de contenido o inquilino.

- [**Publicación**](powershell-syntex-publishing.md)

    Comience aquí si quiere aprender a usar PowerShell para publicar modelos personalizados en bibliotecas de documentos.

- [**Procesamiento**](powershell-syntex-processing.md)

    Comience aquí si desea aprender a usar PowerShell para desencadenar el procesamiento de modelos personalizados en una biblioteca de documentos.

- [**Explicaciones**](powershell-syntex-explanations.md)

    Empiece aquí si quiere aprender a crear una plantilla de explicación personalizada para un modelo de procesamiento de documentos no estructurado mediante PowerShell.
