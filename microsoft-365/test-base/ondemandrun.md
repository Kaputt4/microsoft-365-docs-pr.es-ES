# <a name="run-your-test-on-demand"></a>Ejecución de la prueba a petición

> [!NOTE]
> Test Base ahora proporciona la opción de iniciar una prueba con un enfoque a petición.

## <a name="run-as-request-under-manage-packages"></a>Ejecutar como solicitud en Administrar paquetes

Para un paquete activo, puede acceder a la característica de ejecución en solicitud desde la página Administrar paquetes.

> [!div class="mx-imgBorder"]
> [![Administración de paquetes](Media/runondemand01-managepackages.png) ](Media/runondemand01-managepackages.png#lightbox)

Al especificar el tipo de actualización del sistema operativo y el producto de Windows que están predefinidos con el paquete, puede iniciar la prueba a petición, que se programa inmediatamente para la renovación mensual actual de las actualizaciones de Windows.


> [!div class="mx-imgBorder"]
> [![Ejecución a petición](Media/runondemand02-runonrequest.png) ](Media/runondemand02-runonrequest.png#lightbox)

No es necesario que la prueba se ejecute con su cadencia automática antes de poder usar la característica. Ahora puede decidir qué producto y cuándo se va a probar.

> [!div class="mx-imgBorder"]
> [![Testummary](Media/runondemand03-testsummary.png) ](Media/runondemand03-testsummary.png#lightbox)

> [!NOTE]
> Recuerde que solo los paquetes activos tendrán habilitado el botón Ejecutar al solicitar. Asegúrese de habilitar el paquete para futuras pruebas si desea participar en el paquete para esta característica.
