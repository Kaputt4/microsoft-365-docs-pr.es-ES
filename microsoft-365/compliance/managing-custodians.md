# <a name="work-with-custodians-and-non-custodial-data-sources-in-advanced-ediscovery"></a>Trabajar con administradores y orígenes de datos no administradores en eDiscovery avanzado

Cuando una organización responde a una investigación legal, el flujo de trabajo para identificar, conservar y recopilar contenido potencialmente relevante se basa en las personas de la organización que son los administradores de datos relevantes. En la exhibición de documentos electrónicos, estos individuos se denominan administradores de datos *(o* simplemente administradores) y se definen como "personas que tienen control administrativo de un documento o archivo electrónico". Por ejemplo, el administrador de un mensaje de correo electrónico podría ser el propietario del buzón que contiene el mensaje relevante.

Además, puede haber contenido ubicado en buzones y sitios que no están asociados con un administrador, pero que es relevante para el caso. Las ubicaciones de contenido en las que los administradores de casos no tienen control administrativo pero pueden ser propietarios de datos relevantes, se conocen como orígenes de datos *no administradores.*

En un caso de eDiscovery avanzado, los equipos legales pueden agregar personas de su organización como administradores e identificar y conservar orígenes de datos de custodia, como buzones de Exchange, cuentas de OneDrive y sitios de SharePoint y Teams. También pueden identificar y conservar orígenes de datos que no son de custodia. Al usar la herramienta integrada de administración de orígenes de datos y administradores en eDiscovery avanzado, las organizaciones pueden proteger la información almacenada electrónicamente de la eliminación involuntaria (o intencionada). Esto le permite eliminar el proceso lento y propenso a errores de tener que realizar manualmente los procesos de retención legal.
